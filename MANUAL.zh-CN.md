# 思辨剧场使用说明

## 1. 这是什么

思辨剧场是一个多 AI 辩论与决策辅助网页。

你输入一个问题后，可以让多个 AI，或者同一个 AI 扮演多个不同人格，一起围绕这个问题展开讨论，最后得到：

- 可读的讨论过程
- 联网搜索说明
- 裁判总结
- 可导出的记录

## 2. 什么是全局配置

页面顶部有“全局大模型配置”。

它的意思是：

- 每家厂商的 API Key 只填一次
- 每家厂商的 Base URL 只填一次
- 下方角色卡片只负责选择模型变体

这样更简单，也更不容易填错。

## 3. 什么是 Base URL

Base URL 就是“请求发到哪里去”。

如果你直接在官方购买 API，一般保持默认官方地址即可。  
如果你的 Key 来自第三方中转站，比如 OhMyGPT，就必须把 Base URL 改成中转站提供的地址。

示例：

```txt
https://api.ohmygpt.com/v1
```

## 4. 两种讨论组织方式

- 多模型模式：比较不同厂商模型的观点
- 单 API 多人格模式：同一个模型扮演多个不同角色

## 5. 四种讨论类型

- 定论：尽量给出明确建议
- 分析：允许多个答案并列成立
- 研究：重点在查证、反驳、验证
- 娱乐：重点是有趣和戏剧张力

## 6. 联网搜索说明

- Grok 可以使用原生联网搜索
- DeepSeek 在本项目中使用外部搜索增强
- 如果联网失败，系统会自动降级，不会整场崩掉

## 7. 安全提醒

- 不要把 API Key 截图发给别人
- 不要把 `.env.local` 上传到 GitHub
- 开源仓库里只保留 `.env.example`

## 8. 本地启动

```bash
npm install
npm run dev
```

浏览器打开：

```txt
http://localhost:3000
```

## 9. 免费部署到 Vercel

1. 把项目上传到 GitHub
2. 登录 Vercel
3. 导入你的 GitHub 仓库
4. 点击 Deploy

Vercel 会自动识别这是一个 Next.js 项目，通常不需要你手动配置服务器。
